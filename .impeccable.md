## Design Context

### Users
Engineering stakeholders, operators, and implementation reviewers using this repo as a visual reference for Sterling septic control logic. They need to inspect stage flow, RL relationships, float trigger heights, and actuator meaning quickly without decorative UI getting in the way.

### Brand Personality
Technical, calm, inspectable. The interface should feel precise and trustworthy rather than flashy or promotional. It should support engineering discussion, not try to sell the system.

### Aesthetic Direction
Light-mode, dead minimal, and mobile-first. Favor a narrow reading column, vertical process flow, and the minimum amount of UI needed to explain the system. Avoid visual noise, dashboard chrome, and anything that feels like it is trying to impress.

### Design Principles
1. Make the process flow readable in one simple top-to-bottom pass.
2. Prefer fewer elements over richer framing when clarity is the goal.
3. Keep the layout narrow and mobile-friendly by default.
4. Show only the detail needed to understand the current stage and its trigger points.
5. Remove presentation noise before adding any new visual treatment.
